# Задание: Определение диапазона вставки для поддержания устойчиво отсортированного списка

## Описание задачи

Предположим, что мы разрабатываем структуру данных, которая хранит элементы в отсортированном порядке по определённому ключу.

При поступлении нового элемента необходимо вставить его в список так, чтобы:
1. Список оставался отсортированным.
2. Сортировка была устойчивой — то есть элементы с одинаковым ключом сохраняли порядок добавления:
    - либо новые элементы вставляются после уже существующих с таким же ключом,
    - либо наоборот — перед ними, в зависимости от выбранной политики вставки.

Для реализации такого поведения важно найти диапазон индексов элементов, имеющих заданный ключ, а также ближайшие к ним значения, чтобы определить корректную позицию для вставки.

Этот диапазон позволит:
- определить, где именно можно вставить новый элемент с данным ключом;
- корректно обработать случай, когда элементов с этим ключом нет в списке.

## Цель

Разработать алгоритм, который для заданного ключа возвращает диапазон индексов в отсортированном списке:
- слева — индекс последнего элемента, **меньшего** заданного ключа;
- справа — индекс первого элемента, **большего** заданного ключа.

Этот алгоритм должен быть достаточно эффективным, чтобы использоваться в структуре данных, где вставка новых элементов выполняется часто.

## Основные требования

1. Аналитики:
   - Проанализировать задачу, выделить входные и выходные данные, определить область допустимых значений.
   - Предложить сигнатуру функции и спецификацию возвращаемого результата.
   - Оценить асимптотику эффективного алгоритма для решения задачи.

2. Разработчики:
   - Реализовать алгоритм, обеспечивающий корректное определение требуемого диапазона.
   - Обеспечить эффективность решения и корректную обработку граничных случаев.

3. Тестировщики:
   - На основе постановки задачи и спецификации аналитиков разработать систему тестов для проверки корректности реализации.

## Цель занятия

К моменту завершения занятия должна быть реализована функция, которая корректно определяет диапазон вставки для заданного ключа и проходит все подготовленные тесты.
